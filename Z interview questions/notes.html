let userName: string

  • let is a keyword used to declare a variable.
  • userName is the name of the variable.
  • string is the data type of the variable.

    • Examples:

    let variableName: dataType

    Ex: let userName: string

    in the above example, we are declaring a variable named userName with a data type of string.

    Ex: let userName = "Rahul"

    Data Types in TypeScript

    TypeScript has several built-in data types that can be used to define the type of a variable. Some of the commonly used data types in TypeScript include:

    Types in TypeScript

    TypeScript types define what kind of data a variable can store. They help detect errors during development.

    1. Primitive Types

    Type Description Example
    string Text values "Hello"
    number Numbers (integer or decimal) 10, 5.5
    boolean True or false true, false
    null Empty value null
    undefined Variable declared but not assigned undefined
    bigint Very large numbers 123n
    symbol Unique identifier Symbol()

    Example

            let name: string = "Rahul";
        let age: number = 25;
        let isActive: boolean = true;

    2. Array Type

    Used to store multiple values of the same type.

            let numbers: number[] = [1,2,3,4];
        
        let names: string[] = ["Ram","Shyam","Sita"];

    Alternative syntax:

            let numbers: Array = [1,2,3];

    Advanced array

    Array can also hold multiple types using union types.

            let user: (string | number)[] = ["Hello", 10, "World", 20];
        user = ["John", 25, "Doe", 30];
        user =[10, 20, 30, 40];
        user = ["Alice", "Bob", "Charlie"];

    3. Tuple

    Tuple allows fixed number of elements with specific types.

            let user: [string, number];
        
        user = ["John", 25];

    4. Enum

    Enum defines a set of named constants.

            enum Status {
          Pending,
          Approved,
          Rejected
        }
        
        let orderStatus: Status = Status.Approved;

        enum Role {
          Admin,
          Editor,
          Viewer,
        }

        let userRole: Role = Role.Admin;

    5. Any

    Allows any type of value.

            let data: any = 10;
        
        data = "Hello";
        data = true;

    6. Unknown

    Safer alternative to any.

            let value: unknown = "Hello";

    7. Void

    Used when a function does not return anything.

            function logMessage(): void {
          console.log("Hello");
        }

    8. Object Type

            let user: {name: string, age: number} = {
          name: "Ravi",
          age: 30
        };

    9. Union Type

    Allows multiple possible types.

    we can add more than 2 types n type can be added

            let id: number | string;
        
        id = 10;
        id = "ABC123";

    10. Literal Types

    Restricts variable to specific values.

            let direction: "left" | "right" | "up" | "down";
        
        direction = "left";

    Type alias

    Type aliases allow you to create a new custom type we can call it, if we can use that as custom type

            type userDetail = {
          name: string;
          age: number;
        };
