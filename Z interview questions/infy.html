Playwright + TypeScript Interview Questions & Answers

1. What is Playwright and how is it different from Selenium?
Playwright is an end-to-end testing framework developed by Microsoft. It supports Chromium, Firefox, and WebKit browsers with auto-waiting and faster execution. Unlike Selenium, it does not require external drivers and provides built-in tracing, video, and parallel execution support.
2. What are Browser, Context, and Page in Playwright?
  • Browser: Actual browser instance (Chrome/Firefox/WebKit)
  • Context: Isolated session like incognito window
  • Page: A single tab inside a context
3. What is auto-waiting in Playwright?
Playwright automatically waits for elements to be visible, enabled, and stable before performing actions, reducing the need for manual waits.
4. Difference between locator() and page.$()?
  • locator() supports auto-waiting and is recommended
  • page.$() returns element handle without auto-waiting
5. How do you handle dynamic elements?
Use stable selectors like:
  • getByRole()
  • getByText()
  • data-testid attributes
Avoid fragile XPath selectors.
6. Why use TypeScript with Playwright?
TypeScript provides static typing, better IDE support, fewer runtime errors, and improved maintainability for large automation frameworks.
7. What is Page Object Model (POM)?
POM is a design pattern where each page is represented as a class. It improves reusability, maintainability, and readability of test scripts.
8. Can Playwright be used for API testing?
Yes, Playwright supports API testing using request context: 
const response = await request.get('/api/users');
expect(response.status()).toBe(200);
9. What is Playwright Trace Viewer?
It is a debugging tool that shows actions, DOM snapshots, network calls, and screenshots to analyze test failures.

11. Can Playwright be used for API testing?
Yes, Playwright supports API testing using its request context. You can send GET, POST, PUT, DELETE requests and validate responses. 
const response = await request.get('/api/users');
expect(response.status()).toBe(200);
12. What is Playwright Trace Viewer?
Trace Viewer is a debugging tool that helps analyze test failures. It shows screenshots, DOM snapshots, network requests, and actions step-by-step. It is very useful for identifying flaky tests and root causes of failures.
13. How do you run tests in parallel in Playwright?
Playwright runs tests in parallel by default using workers. You can also configure it in the config file. 
workers: 4
This improves execution speed significantly.
14. How do you handle flaky tests?
To handle flaky tests:
  • Use stable locators like data-testid
  • Avoid hard waits (setTimeout)
  • Use Playwright auto-waiting features
  • Use trace viewer for debugging
  • Avoid dependency between tests
15. How do you mock API responses in Playwright?
You can intercept network requests using page.route(). 
await page.route('**/api/users', route =>
  route.fulfill({
    status: 200,
    body: JSON.stringify([{ name: 'John' }])
  })
);
16. What are fixtures in Playwright?
Fixtures are used to set up and tear down test environments. They help share common setup logic like browser, page, or login state across tests.
17. How do you capture screenshots and videos?
You can configure this in Playwright config file. 
use: {
  screenshot: 'only-on-failure',
  video: 'retain-on-failure'
}
This helps in debugging failed tests.
18. What is test isolation in Playwright?
Test isolation means each test runs independently in a separate browser context. This prevents shared state like cookies or sessions from affecting other tests.
19. How do you debug tests in Playwright?
You can debug using:
  • Playwright Inspector
  • Trace Viewer
  • Headed mode execution
  • Console logs and screenshots
20. Why is Playwright preferred over other tools?
Playwright is preferred because:
  • Fast execution
  • Auto-waiting reduces flakiness
  • Cross-browser support
  • Built-in parallel execution
  • Powerful debugging tools

Data Types in TypeScript

TypeScript has several built-in data types that can be used to define the type of a variable. Some of the commonly used data types in TypeScript include:

Types in TypeScript

TypeScript types define what kind of data a variable can store. They help detect errors during development.

1. Primitive Types

Type Description Example
string Text values "Hello"
number Numbers (integer or decimal) 10, 5.5
boolean True or false true, false
null Empty value null
undefined Variable declared but not assigned undefined
bigint Very large numbers 123n
symbol Unique identifier Symbol()

Example

                  let name: string = "Rahul";
                  let age: number = 25;
                  let isActive: boolean = true;

2. Array Type

Used to store multiple values of the same type.

                  let numbers: number[] = [1,2,3,4];
                  
                  let names: string[] = ["Ram","Shyam","Sita"];

Alternative syntax:

                  let numbers: Array = [1,2,3];

Advanced array

Array can also hold multiple types using union types.

                  let user: (string | number)[] = ["Hello", 10, "World", 20];
                  user = ["John", 25, "Doe", 30];
                  user =[10, 20, 30, 40];
                  user = ["Alice", "Bob", "Charlie"];

3. Tuple

Tuple allows fixed number of elements with specific types.

                  let user: [string, number];
                  
                  user = ["John", 25];

4. Enum

Enum defines a set of named constants.

                  enum Status {
                    Pending,
                    Approved,
                    Rejected
                  }
                  
                  let orderStatus: Status = Status.Approved;
          
                  enum Role {
                    Admin,
                    Editor,
                    Viewer,
                  }
          
                  let userRole: Role = Role.Admin;

5. Any

Allows any type of value.

                  let data: any = 10;
                  
                  data = "Hello";
                  data = true;

6. Unknown

Safer alternative to any.

                  let value: unknown = "Hello";

7. Void

Used when a function does not return anything.

                  function logMessage(): void {
                    console.log("Hello");
                  }

8. Object Type

                  let user: {name: string, age: number} = {
                    name: "Ravi",
                    age: 30
                  };

9. Union Type

Allows multiple possible types.

we can add more than 2 types n type can be added

                  let id: number | string;
                  
                  id = 10;
                  id = "ABC123";

10. Literal Types

Restricts variable to specific values.

                  let direction: "left" | "right" | "up" | "down";
                  
                  direction = "left";

Type alias

Type aliases allow you to create a new custom type we can call it, if we can use that as custom type

                  type userDetail = {
                    name: string;
                    age: number;
                  };

Most Used Types in Real Projects

  • string
  • number
  • boolean
  • Array
  • Object
  • Union
